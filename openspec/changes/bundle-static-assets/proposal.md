# Proposal: 将静态资源打包进二进制执行文件

## Why
目前 `air` 在运行时依赖外部文件 `docs/favicon.svg`。如果用户将编译后的二进制文件移动到其他目录执行，程序将无法找到该图标并报错。作为一个便携式的 CLI 工具，`air` 应该尽可能减少对外部非共享文件的依赖。

## What Changes
- 使用 Rust 的 `include_bytes!` 宏将 `docs/favicon.svg` 的内容在编译时嵌入到二进制文件中。
- 修改 `src/handlers.rs` 中的 `favicon` 处理函数，使其直接从内存中读取并返回图标内容，而不是从磁盘读取。

## Impact
- **便携性提升**：单文件二进制即可完整运行，无需携带额外的 `docs/` 目录。
- **鲁棒性**：消除了因磁盘文件丢失或路径不正确导致的图标显示失败。
