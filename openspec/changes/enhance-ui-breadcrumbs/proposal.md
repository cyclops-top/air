# Proposal: 优化 Web UI 路径展示为可点击的面包屑

## 摘要
目前 `air` 的 Web UI 使用简单的 "Current: /path" 字符串展示当前位置。本提案旨在将其改进为可点击的面包屑导航（Breadcrumbs），提升用户在目录间切换的体验。

## 目标
1.  **路径可点击**：路径中的每一个层级都应该是可点击的链接，方便回溯到任意父级目录。
2.  **视觉美化**：移除 "Current: " 前缀，使用更现代的分割符和悬停效果。
3.  **根路径映射**：第一个层级固定映射为 "Home" 或 "根目录"。

## 范围
*   **view.rs**: 修改 `render_html` 逻辑，增加面包屑生成算法。
*   **CSS**: 增加面包屑相关的样式（如链接颜色、分隔符间距、悬停背景等）。
