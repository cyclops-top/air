# Proposal: 实现全屏 TUI 仪表盘

## 摘要
目前的 `air` 工具使用传统的命令行滚动输出。本提案旨在引入全屏 TUI（终端用户界面）模式，提供一个更现代、直观的运行状态仪表盘。仪表盘将固定显示服务器信息，并提供一个可滚动的实时日志区域。

## 目标
1.  **全屏体验**：启动后进入全屏 TUI 模式。
2.  **固定头部**：在屏幕顶部固定显示共享路径、安全状态、访问地址等基本信息。
3.  **动态日志**：在剩余空间显示实时请求日志，支持滚动浏览。
4.  **优雅退出**：按下 'q' 键退出 TUI 并停止服务器。
5.  **离线统计**：TUI 退出后，在终端控制台打印本次运行的下载统计摘要。

## 范围
*   **构建配置**: 添加 `ratatui` 和 `crossterm` 依赖。
*   **状态管理**: 在 `Stats` 中增加线程安全的日志队列。
*   **日志系统**: 修改 `logger.rs`，将日志从直接打印改为推送到队列。
*   **UI 实现**: 新增 `dashboard.rs` 负责 TUI 渲染和输入处理。
*   **主流程**: 修改 `main.rs` 协调服务器运行与 TUI 循环。
