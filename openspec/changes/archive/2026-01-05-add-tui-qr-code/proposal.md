# Proposal: TUI 增加二维码扫码分享

## Why
目前在局域网分享文件时，如果接收方是手机，用户需要手动在手机浏览器输入长串的 IP 地址和端口号，体验较差。在终端 TUI 界面直接展示二维码，可以极大简化手机访问的流程。

## What Changes
- **依赖引入**: 添加 `qrcode` crate 用于生成二维码数据。
- **UI 更新**: 在 `src/dashboard.rs` 的 Header 区域增加一个子布局，用于并排显示文字信息和二维码。
- **动态生成**: 根据当前的局域网 IP、端口和协议（HTTP/HTTPS）动态生成指向 Network 地址的二维码。
- **渲染适配**: 使用 Unicode 字符（如 `█`）在终端中渲染二维码。

## Impact
- **易用性**: 手机用户只需“扫一扫”即可访问，无需手动输入。
- **视觉效果**: 提升 TUI 界面的专业感和现代感。
