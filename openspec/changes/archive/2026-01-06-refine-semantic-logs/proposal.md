# Change: 精细化语义日志与 UI 对齐优化

## Why
目前的日志虽然有了语义化标签，但在 TUI 界面中缺乏颜色区分，且由于路径长度不一导致列对齐混乱。通过引入颜色、固定列宽、过滤冗余请求（如 Favicon）以及调整显示顺序（耗时前置），可以极大地提升 TUI 仪表盘的专业感和实用性。

## What Changes
- **过滤冗余**: 停止记录 `FAVICON` 相关的日志。
- **重构展示格式**:
    - 移除 HTTP 状态码。
    - 将“耗时”移至“路径”之前。
    - 采用固定宽度的列：`[时间] [IP] [动作] [耗时]`。
- **色彩与状态语义**:
    - 为 `DOWNLOAD` 和 `OPEN DIR` 添加颜色区分（例如：蓝色与绿色）。
    - 针对请求失败（非 200/206）：路径前缀增加 `✖`，且路径整体显示为红色。
- **对齐与截断**:
    - 如果日志行过长，对路径进行 `...` 缩写截断，确保不换行。
- **数据结构升级**: 将 `Stats` 中的日志存储从 `String` 改为结构化的 `LogEntry`，以便在 TUI 渲染层进行精细化样式控制。

## Impact
- **TUI 界面**: 日志区域将变得井然有序，颜色对比能让用户迅速捕捉关键操作。
- **代码结构**: `LogAction` 枚举将移动至 `handlers.rs` 以支持结构化存储。
