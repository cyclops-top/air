# Proposal: 实现 HTTPS 支持与 HTTP/2 性能优化 (可配置)

## Why
目前 `air` 仅支持 HTTP/1.1，在局域网环境下由于浏览器并发连接数限制（通常为 6 个），当页面请求大量小文件或频繁获取目录信息时，性能会受到显著影响。
通过启用 HTTPS，我们可以开启 HTTP/2 的多路复用（Multiplexing）特性，从而在单一 TCP 连接上并发处理大量请求，极大提升“批量请求”的性能。针对局域网 IP 环境，我们将采用自动生成自签名证书的方案。

## What Changes
- **CLI 增强**：添加 `--https` 命令行参数，允许用户选择开启 HTTPS 模式。
- **证书自动化**：集成 `rcgen`，在 HTTPS 模式开启时自动生成针对当前局域网 IP 的自签名证书。
- **HTTPS 服务器**：集成 `axum-server` 以支持 TLS 监听。
- **H2 优化**：显式配置 Axum 开启 HTTP/2 多路复用。
- **TUI 更新**：在全屏仪表盘中显示当前的协议模式 (HTTP 或 HTTPS)。

## Impact
- **性能**：高并发请求下的加载速度显著提升。
- **安全性**：提供可选的传输层加密。
- **兼容性**：默认仍为 HTTP，确保向后兼容。
