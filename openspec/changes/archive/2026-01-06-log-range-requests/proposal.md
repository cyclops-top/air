# Proposal: 在日志中记录分段下载的 Range 信息

## Why
目前对于大文件的分段下载（HTTP Range Requests），TUI 仪表盘会产生大量完全相同的日志条目（显示为 `DOWNLOAD` 和 路径），用户无法区分这些请求是重复下载还是正常的流式/分段下载。通过在日志中标记具体的 `Range` 信息，可以清晰地展示文件下载的进度和分段情况。

## What Changes
- **扩展日志模型**: 在 `LogEntry` 结构体中增加一个可选的 `range` 字段。
- **提取 Range 头部**: 修改 `src/logger.rs` 中的日志中间件，从 HTTP 请求头中提取 `Range` 字段。
- **UI 展示增强**: 修改 `src/dashboard.rs` 的渲染逻辑，如果请求包含 `Range` 信息，则在动作标签后或路径前以小字/括号形式展示（例如：`DOWNLOAD [0-1023]`）。
- **去重/去干扰**: (可选) 确保只有有效的 Range 信息被记录。

## Impact
- **可观测性**: 用户可以直观看到视频流播放或多线程下载时的详细分段行为。
- **调试友好**: 方便排查某些分段请求是否被正确处理。
