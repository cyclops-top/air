# Proposal: 使用 Rust 实现 Air 核心功能

## 摘要
本提案旨在基于 `docs/technical-specification.md` 提供的技术规范，使用 Rust 语言从零开发 `air` 命令行工具。该工具将提供轻量级、安全的局域网文件共享服务，支持断点续传和极简 Web UI。

## 目标
1.  **高性能与安全性**：利用 Rust 的内存安全特性和类型系统，确保文件服务的稳定与安全。
2.  **符合规范**：严格遵循技术文档中关于 CLI 参数、日志格式、OpenAPI 接口定义以及安全沙箱的要求。
3.  **标准化构建**：使用 `Makefile` 管理构建、测试和运行流程。

## 范围
*   **CLI 交互**：参数解析、启动 Banner、运行时日志。
*   **HTTP 服务**：静态文件服务、目录列表渲染、Range 请求支持。
*   **安全机制**：路径防穿越检查、隐藏文件过滤。
*   **构建系统**：Makefile 集成 Cargo 命令。
